# ТЗ: Редизайн и развитие рас — Chaos Addon (Origins, NeoForge 1.21.1)

> Документ составлен по итогам прямого аудита кода и JSON-данных репозитория (23 расы, 182 файла способностей, ~9800 строк Java). Все находки в части 1 проверены построчно — это не предположения, а факты из кода.

---

## ЧАСТЬ 1. Нелогичности и то, что реально мешает играть

### 1.1. Дублирующиеся способности внутри ОДНОЙ расы
- **Странник Времени** тратит 2 из 8 слотов способностей на практически одинаковый эффект: `stasis_field` («заморозить всех в радиусе 15 блоков на 8 сек, двойной урон по замороженным, сам тоже не двигаешься») и `time_loop` («заморозить всех в радиусе 15 блоков на 10 сек + Confusion, двойной урон по замороженным»). Игрок получает массовый стан дважды под разными названиями вместо двух разных инструментов.

### 1.2. Copy-paste способностей между НЕСВЯЗАННЫМИ расами без переосмысления
- `sun_burn` («солнце наносит 0.5❤ каждые 5 сек») дословно одинаков у **Пожирателя Миров** и **Лунного Отступника** — двух тематически разных рас.
- `time_loop` («заморозить всех + урон х2 замороженным») есть и у **Странника Времени**, и у **Пожирателя Миров** — с одинаковым названием, описанием и числами. Race-флейвор теряется: два «времени» в моде делают одно и то же.

### 1.3. Стакинг дебаффов без «клапана сброса» — расы, в которые объективно тяжело/неприятно играть
- **Древний Страж**: −50% скорости, прыжок максимум 1 блок, нет элитры, нет лодок, нет зелий скорости, **+ мгновенное утопление в воде/лаве**. Это пять разных ограничений мобильности одновременно на одной расе. По факту — раса не может ни плавать, ни летать, ни быстро ходить, ни плавать на лодке с друзьями. На любой карте с океаном или большими расстояниями это не «медленный танк», а «застрял дома».
- **Мицелиальный Симбионт**: привязан к мху (не дальше 20 блоков, иначе урон) + дождь и вода **уничтожают сам мох**. На сервере, где дождь идёт регулярно и его нельзя контролировать, эти два дебаффа взаимно усиливают друг друга — раса может попасть в петлю «мох смыло → нет привязки → урон без возможности исправить ситуацию, пока не закончится дождь».
- **Глубинный Навигатор**: лечится только проходом через портал (+1❤ за проход) **и одновременно** теряет HP каждые 30 сек, если портала нет в радиусе 100 блоков. Если игрок исследует мир вдали от порталов — у него физически нет способа восстановить здоровье, кроме как идти обратно. Это вынуждает либо никогда не отходить от портала, либо строить персональный портал у дома и бегать сквозь него туда-обратно ради лечения — нездоровый, вырожденный геймплейный паттерн (не приключение, а абьюз механики).
- **Инфернальный Пастух**: смерть в Обычном мире = весь инвентарь теряется без возможности вернуть. Без какого-либо шанса на откуп (тотем, страховка, частичное сохранение) — один неудачный респавн на чужой территории может стоить игроку часов прогресса.

### 1.4. Способности, которые в описании заявлены как «реализовано Java», но в коде их не существует

Ещё ~34 способности (видение сквозь стены, «нельзя носить оружие/кирки/шлем», «часы взрываются», «алмазы исчезают из рук» и т.д.) — либо чистый текст без механики, либо реализованы через нулевой `attribute_modifier` (amount: 0.0), который физически ничего не меняет. Это значит: **заявленные минусы рас часто фейковые**, а заявленные плюсы (активные способности через команды) почти все реально работают — то есть фактический баланс ещё сильнее перекошен в сторону силы, чем кажется по описаниям в игре.

### 1.5. Дублирование «личности» расы
**Паразитический Разум** и **Нейро-Захватчик** — это, по сути, одна и та же раса, выпущенная дважды: заразить слабую цель → отдавать приказы марионетке (макс. 3) → взорвать её по команде → ноль брони/прямого урона. Различия (кража эффектов, синхронная атака) косметические. Из 23 рас две по факту дают одинаковый игровой опыт.

### 1.6. Несоответствие текста и механики (вводит игрока в заблуждение)
**Повелитель Роя**, способность `no_helmet`: текст гласит «**не может** носить шлем». Реальная механика — шлем можно физически надеть, просто он не даёт брони. Это разные вещи: один вариант — хард-запрет, другой — скрытая бесполезность предмета, которую игрок обнаружит только потеряв здоровье в бою.

## ЧАСТЬ 2. Ориентир: чем силён оригинальный Origins

Ванильный Origins держится на правиле «**одна яркая фишка + один моментально читаемый минус**» (Avian — двойной прыжок ценой шлема и двойного урона от падения; Merling — суперплавание ценой урона на суше). У Chaos Addon контента на расу в 1.5–2 раза больше — это плюс по объёму, но без UI-поддержки и без вышеописанных багов получается противоположный эффект: вместо «понятно с первого взгляда» — «нужно гуглить чужой текстовый файл, чтобы понять, что вообще происходит».

## ЧАСТЬ 3. 50 улучшений, усилений, переработок и переосмыслений

### A. Системные изменения (затрагивают все 23 расы сразу)

4. **Реализовать все 39 «текстовых» способностей по-настоящему** либо честно убрать их из описаний — текущее состояние обманывает игрока.
6. **Задействовать GeckoLib и Pehkui**, которые уже лежат в зависимостях, но не используются нигде в коде — кастомные анимированные модели для трансформационных рас (Биоморф, Зеркальный Фантом, Некровор) и масштабирование тела (рост/уменьшение) вместо одних частиц.
7. **Балансировочный конфиг для всех рас без исключения** — сейчас 5 новейших рас (Necrovore, Neural Hijacker, Blood Smith, Star Oracle, Mirror Phantom) не настраиваются без правки Java.
8. **Серверная команда `/chaos balance-report`** — автоматический вывод текущих win-rate/usage-метрик по расам (если есть статистика смертей/убийств), чтобы решения по балансу принимались по данным, а не на глаз.
9. **Унифицировать «клапаны сброса» для тяжёлых дебаффов.** Любой стакающийся минус (как у Стража или Симбионта) должен иметь явный безопасный выход — таймер отката, временный иммунитет, предмет-страховку.
10. **Чистка дублей способностей** (`sun_burn`, `time_loop` копипаст между расами; `stasis_field`/`time_loop` дубль внутри Странника Времени) — на их место можно поставить новые, уникальные эффекты вместо повторов.

### B. Точечные переработки проблемных рас

11. **Древний Страж — снять «пятислойный» паралич мобильности.** Оставить медлительность и отсутствие прыжка как ядро фишки «гора-переросток», но вернуть лодки (или дать расе личный аналог «каменного плота») и заменить мгновенное утопление на постепенный урон с возможностью выплыть.
12. **Мицелиальный Симбионт — защита моха от дождя под крышей/блоками.** Дать механику «грибной зонтик» (блок укрытия, защищающий локальный мох от воды) вместо безусловного уничтожения сети любым дождём.
13. **Глубинный Навигатор — альтернативный источник лечения.** Добавить медленную пассивную регенерацию вдали от портала (вдвое медленнее обычной) вместо чистого урона по таймеру — убирает вырожденный «бег сквозь портал туда-обратно».
14. **Инфернальный Пастух — частичная страховка при смерти в Обычном мире.** Например: 1 предмет можно «обречь на спасение» заранее (зачаровать), он не теряется.
15. **Странник Времени — слить `stasis_field` и `time_loop` в одну сильную способность**, а освободившийся слот отдать под новую: например, «Отголосок» — кратковременная копия себя из прошлого, повторяющая твои действия 5-секундной давности как отвлекающий маневр.
16. **Пожиратель Миров и Лунный Отступник — разные версии `sun_burn`.** У Пожирателя — урон растёт с количеством хаос-эффектов вокруг (тематика энтропии); у Лунного — урон пропорционален фазе луны прошлой ночи (тематика цикличности).
17. **Парасайт-дуэт — развести Паразитический Разум и Нейро-Захватчика по разным осям геймплея.** Например: Паразитический Разум — про массовость и количество марионеток (рой из мелких заражённых), Нейро-Захватчик — про качество и один сильный «носитель» с украденными способностями этого конкретного моба.
18. **Судья Измерений — пересчитать стоимость силы.** «Аннигиляция» (гарантированное убийство) и «Балансировка» (массовый хил/контроль команды) на коротких по меркам их мощи откатах — добавить реальную цену: например, «Аннигиляция» не убивает мгновенно, а отнимает процент текущего HP цели, что делает её опаснее против слабых и менее опасной против боссов/танков, сохраняя при этом риск для самого Судьи.
19. **Повелитель Роя — настоящий хард-запрет на шлем**, соответствующий тексту, либо переписать текст под реальную механику («шлем технически не убирается физически, но действует как пустой слот»).
20. **Фантомный Археолог — реальная, а не фейковая привязка к элитрам.** Раз тема расы — «жизнь в облаке/чанк-данные», логичнее дать **бонус**, а не вычеркнутый минус: телепорт между загруженными чанками вместо полёта элитрой, что более тематично и реально работает.
21. **Алхимический Монах — смягчить «штраф за инвентарь».** Сейчас и `no_armor`, и `overloaded_damage` совместно дают +30% получаемого урона при переполненном инвентаре поверх отсутствия нормальной брони — двойное наказание за обычный лутинг. Сделать порог индивидуальным для предметов категории «реагенты», а не для любого стака.
22. **Инженер Хаоса — смягчить тройную зависимость от редстоуна.** Сейчас и еда не работает без редстоуна, и урон удвоен без него в инвентаре — фактически нужно вечно таскать стак красной пыли как «кислородный баллон». Снизить до одного жёсткого требования вместо двух одновременно.
23. **Звёздный Провидец — убрать дублирующий текст `cosmic_body`/`melee_vulnerability`.** Свести в одну способность с одним внятным описанием, высвободив слот под новую активку (например, «Затмение взгляда» — кратковременная слепота себе в обмен на удар по всем видящим тебя врагам).
24. **Кровавый Кузнец — визуальный индикатор зарядов крови** прямо над хотбаром/в виде ауры на теле, а не только в action bar — повышает читаемость ключевого ресурса расы.
25. **Странствующий Садовник — дать настоящую альтернативу оружию.** Сейчас бой полностью завязан на приручение мобов едой; добавить «шипастую лозу» как тематическое оружие-замену (атака растениями вместо стали), а не полный ноль урона в ближнем бою.

### D. Техническая инфраструктура под будущее развитие рас

41. **CI-проверка целостности данных при каждом коммите** — автоматический прогон скрипта сверки ID способностей (как сделанный вручную в этом аудите), падающий билд при несовпадении.
42. **Серверная проверка кулдауна и происхождения прямо в каждой Brigadier-команде**, а не только на стороне JSON-способности — закрывает дыру, когда 16 из 27 команд можно вызвать в чате напрямую без расы и без отката.
43. **Общий хелпер `OriginGuard.requireOrigin(player, originId)`** для всех команд — единая точка проверки вместо разрозненного кода, где часть команд проверку забыли.
44. **Логирование вместо немого `catch (Exception e) { return false; }`** в `OriginHelper` — без этого баги вроде опечатки `balance_of_powers` остаются невидимыми годами.
45. **Юнит-тесты на JSON-схему** (обязательные поля, существование power-файлов для каждого ID из origin-списка, отсутствие «осиротевших» файлов вроде `no_elytra`).
46. **Вынести магические числа из Java в конфиг** для всех рас без исключения — единый источник правды для баланса, а не хардкод вперемешку с конфигом для части рас.
47. **README.md с игровой документацией** — отдельно от `replit.md` (это дев-заметка для агента, не гайд для игроков/контрибьюторов).
48. **Чистка репозитория**: нормальный `.gitignore` под Java/Gradle (`build/`, `.gradle/`, `*.class`), вынос `attached_assets/` (чужие jar, краш-логи, скриншоты — 26 МБ мусора) из основной ветки, удаление случайно попавшего JS/TS-шаблона (`package.json`, `pnpm-lock.yaml`, `scripts/`) — сейчас репозиторий весит 710 МБ, должен весить единицы МБ.
49. **Версионирование и changelog по расам** — при таком объёме контента (23 расы × 8 способностей) без построчного changelog невозможно понять, что изменилось между `3.0.0` и `3.0.1`, кроме как читать дифф каждого коммита вручную.
50. **Песочница для тестирования баланса** — отдельный командный режим (`/chaos testmode`), временно выдающий любую расу с обнулёнными кулдаунами для быстрой проверки изменений без пересборки мира.
